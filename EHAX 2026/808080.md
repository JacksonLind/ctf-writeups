## Challenge Overview

The challenge presents a web interface at `chall.ehax.in:8076` titled **#808080** — grey in hex RGB. Players are given a 145-bit binary string and a clickable 32-slice, 5-ring wheel. The goal is to configure the wheel correctly and decode the binary input to reveal the flag.

The challenge hint says: *"gray code? nah grey code"* — pointing towards the wheel needing to be configured as a Grey code table.

## Understanding the Mechanism

The web page source code reveals the exact decode logic:

1. The 145-bit binary input is split into 29 five-bit chunks.
2. For each chunk value `C`, the server searches the wheel for the slice index `i` where `wheelData[i] == C`.
3. That slice index `i` is used to look up the server's internal character table, producing one output character.
4. The wheel is submitted as an array of 32 five-bit strings (one per slice, inner ring = MSB).

### Wheel Validation Rules

The `/api/validate-wheel` endpoint enforces two rules:
- All 32 wheel elements must be **unique** (a permutation of 0–31).
- **Adjacent elements** (including wrap-around 31→0) must differ by exactly 1 bit.

These are precisely the properties of a valid Grey code sequence.

## Step 1: Probe the API Directly

Rather than clicking the wheel manually, we call the API from the browser console. First, try the standard reflected Grey code wheel:

```javascript
const w = [];
for (let i = 0; i < 32; i++) {
  const gray = i ^ (i >> 1);
  w.push(gray.toString(2).padStart(5, '0'));
}
const r = await fetch('/api/decode', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ binaryInput: '<binary>', wheelData: w })
});
console.log(await r.json());
// Result: { decoded: 'GXJBKVI_DGSHI&KSIWAZIV_AL}VHC' }
```

This validates successfully but the output is garbled — the standard wheel is not the intended solution.

## Step 2: Reverse-Engineer the Character Table

With the standard Grey code wheel, each 5-bit chunk `C` maps to character index `gray_inv(C)`. The decoded output reveals the server's character table:

```
 0→A   1→B   2→C   3→D   4→E   5→F
 6→G   7→H   8→I   9→J  10→K  11→L
12→M  13→N  14→O  15→P  16→Q  17→R
18→S  19→T  20→U  21→V  22→W  23→X
24→Y  25→Z  26→@  27→#  28→_  29→{
30→}  31→&
```

Positions 0–25 are A–Z. Positions 26–31 are special characters: `@ # _ { } &`

## Step 3: Try Rotated Grey Code Wheels

A rotated Grey code shifts all slice assignments: `wheelData[i] = gray((i + offset) % 32)`. We test all 32 rotations:

```javascript
for (let offset = 0; offset < 32; offset++) {
  const w = [];
  for (let i = 0; i < 32; i++) {
    const g = (i + offset) % 32;
    w.push((g ^ (g >> 1)).toString(2).padStart(5, '0'));
  }
  const r = await fetch('/api/decode', { ...body: { binaryInput, wheelData: w } });
  console.log(offset, (await r.json()).decoded);
}
```

**Offset 4** produces the flag:

```
offset=0: GXJBKVI_DGSHI&KSIWAZIV_AL}VHC
offset=4: CTF{GREY&CODE#GOES_VERY_H@RD}  <-- FLAG!
```

## Full Solution Script

```javascript
// Offset-4 rotated Grey code wheel
const w = [];
for (let i = 0; i < 32; i++) {
  const g = (i + 4) % 32;
  w.push((g ^ (g >> 1)).toString(2).padStart(5, '0'));
}

// Validate
const v = await fetch('/api/validate-wheel', {
  method: 'POST', headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ wheelData: w })
});
console.log('Validation:', await v.json());

// Decode
const d = await fetch('/api/decode', {
  method: 'POST', headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    binaryInput: '0010111100011010000101111111110110010010000100010111011001000110010000011111101101100111010000010101011001111110010000000111010001111110010000011',
    wheelData: w
  })
});
console.log('Flag:', (await d.json()).decoded);
// Output: CTF{GREY&CODE#GOES_VERY_H@RD}
```

## Key Insights

- The title `#808080` is grey in RGB hex — hinting the wheel must be a Grey code configuration.
- The wheel validation checks the two defining properties of a Grey code: all values unique, adjacent values differ by exactly 1 bit.
- Many valid Grey codes exist. The correct one is a **rotation-by-4** of the standard reflected Grey code.
- The character table at positions 26–31 uses non-standard special characters (`@ # _ { } &`), which appear literally in the flag.
- Calling the API directly from the browser console bypasses the need to manually click 160 wheel cells.
